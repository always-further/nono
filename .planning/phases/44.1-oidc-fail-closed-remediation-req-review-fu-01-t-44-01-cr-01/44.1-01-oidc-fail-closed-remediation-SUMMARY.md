### Task 2 — GREEN gate: implemented the helper + wired both verify callsites

Added the helper `fn read_required_oidc_issuer(user_issuer: Option<&str>) -> std::result::Result<String, String>` above `verify_multi_subject_file` in `crates/nono-cli/src/trust_cmd.rs`. Helper logic:

1. If `user_issuer` is `Some(s)` → return `Ok(s.to_string())` (explicit flag wins).
2. Else read `NONO_TRUST_OIDC_ISSUER`; if unset / whitespace-only → return canonical fail-closed error (`"keyless bundle requires --issuer <OIDC_URL> or NONO_TRUST_OIDC_ISSUER env-var (exact match against signer's iss claim)"`).
3. Else `url::Url::parse` the env-var value; on parse failure return a structured error naming both the env-var and "is not a valid URL".
4. Else return the parsed env-var string.

Both keyless verify callsites at `verify_multi_subject_file:~976` and `verify_single_file:~1172` were swapped to call `read_required_oidc_issuer(user_issuer)?` instead of `trust::signing::configured_oidc_issuer()`. Added `use url::Url;` to the import block.

GREEN gate confirmed: all 5 helper tests pass (`cargo test -p nono-cli --bin nono trust_cmd::tests::read_required_oidc_issuer` = 5 passed, 0 failed); `cargo build -p nono-cli` clean.

### Task 3 — Removed dead library function + regression-codifying tests

Deleted `pub fn configured_oidc_issuer()` and its doc block from `crates/nono/src/trust/signing.rs` (lines 136-193 in pre-44.1 state). After Task 2 the function had zero non-test callers in the workspace — CLAUDE.md § "Lazy use of dead code" forbids retention. Replaced with an inline comment block documenting the deletion rationale and pointing readers to the new CLI-side helper.

Deleted the 4 unit tests + `OidcEnvGuard` + `OIDC_ENV_LOCK` scaffolding (lines 1107-1205 in pre-44.1 state). Crucially this includes the **regression-codifying** test `configured_oidc_issuer_falls_back_to_github_default_when_unset` whose assertion was the implementation contract for the T-44-01 regression. Coverage for the malformed-env, whitespace-only, and env-set-returns-env-value branches all migrated to corresponding tests in `crates/nono-cli/src/trust_cmd.rs::tests`.

`pub const GITHUB_ACTIONS_OIDC_ISSUER` retained — still used by `test_github_id_token_*` tests and as a documented pin reference for the updated CLI doc. `validate_oidc_issuer` (URL-component equality primitive) retained — Phase 44.1 only removes the silent-default-substitution behavior, not the equality-check primitive.

### Task 4 — Updated CLI `--issuer` doc string

Updated `crates/nono-cli/src/cli.rs:3046-3049` to explicitly document the restored contract: `--issuer` is REQUIRED unless `NONO_TRUST_OIDC_ISSUER` is explicitly set; no implicit default trust anchor exists. The previous doc text said "REQUIRED" but the implementation had silently substituted a default, contradicting the documentation — the doc fix removes that contradiction.
