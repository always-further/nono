## session.rs Deviation — Confirmation

The `session.rs:410` `sessions_dir()` callsite preserves the legacy `NonoError::ConfigParse("Cannot determine home directory for session registry")` error message via `.map_err(|_| ...)`. Verified post-migration:

```bash
$ grep -A4 'pub fn sessions_dir' crates/nono-cli/src/session.rs
pub fn sessions_dir() -> Result<PathBuf> {
    let home = crate::config::nono_home_dir().map_err(|_| {
        NonoError::ConfigParse("Cannot determine home directory for session registry".to_string())
    })?;
    Ok(home.join(".nono").join("sessions"))
```

Both gates pass:
- `nono_home_dir().map_err` present
- `ConfigParse` error variant preserved

This means a non-absolute `NONO_TEST_HOME` will be reported from this callsite as `ConfigParse("Cannot determine home directory for session registry")` rather than the more specific `EnvVarValidation` — intentional, per Plan Edit 1.8 rationale (preserves contract for any caller/test matching on the legacy string). The `EnvVarValidation` error still fires at the 14 other callsites and at the dedicated unit tests in Plan 01.

## Decisions Made

1. **Display-style sites use `if let Ok(home) = ...` not `if let Some(home) = ....ok()`** — Plan called for `.ok()`, but newer clippy (rust-1.95) flags it as `match_result_ok` redundancy with `-D warnings` strict. The plan-locked guidance allowed the `.ok()` form because it was assumed clippy-clean; the suggested fix (`if let Ok(home) = ...`) is semantically identical and clippy-clean. Documented as Rule 3 deviation (auto-fix blocking issue).

2. **session.rs ConfigParse preserved** — per Edit 1.8 explicit deviation guidance.

3. **profile::home_dir() wrapper kept** — body replaced rather than wrapper deleted, to minimize churn (~30 internal callsites in profile/mod.rs continue using the local name).

4. **#[allow(dead_code)] removed from nono_home_dir() and warn_once_test_home()** — Plan 01's Decision 1 explicitly tracked this for cleanup once consumers landed (Plan 02). Both functions are now live across 11 src files.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Display-style sites use `if let Ok(home) = ...` instead of `if let Some(home) = ....ok()`**

- **Found during:** Task 2 (clippy verification after initial Edit 2.1-2.4 application)
- **Issue:** Plan called for `if let Some(home) = crate::config::nono_home_dir().ok() { ... }` and asserted (line 506 of plan) "no clippy warning". In practice, `cargo clippy -p nono-cli --no-deps -- -D warnings` on rust-1.95 flagged this as `clippy::match_result_ok`: *"matching on `Some` with `ok()` is redundant"*. With `-D warnings` strict (CLAUDE.md mandate), this blocked the build.
- **Fix:** Applied clippy's suggested fix — `if let Ok(home) = crate::config::nono_home_dir() { ... }`. Semantically identical: still silently swallows any `Err(EnvVarValidation/HomeNotFound)`, still returns the unchanged input on miss. The plan's stated intent (preserve display-only Option-shape semantics, no `?`-propagation) is fully preserved.
- **Files modified:** `audit_commands.rs`, `rollback_commands.rs` (2 sites), `trust_scan.rs` (2 sites)
- **Verification:** `cargo clippy -p nono-cli --no-deps -- -D warnings -D clippy::unwrap_used` clean on those files. `cargo test -p nono-cli --bin nono rollback_commands::tests::shorten_home_replaces_prefix` passes.
- **Committed in:** `1b4e7343` (Task 2 commit)

**2. [Rule 2 - Missing Critical] Removed `#[allow(dead_code)]` from `nono_home_dir()` and `warn_once_test_home()`**

- **Found during:** Task 3 (Cargo.toml dep removal)
- **Issue:** Plan 01 explicitly added `#[allow(dead_code)]` because consumers were deferred to Plan 02 (its Decision 1 also flagged "Plan 02 task should remove the #[allow(dead_code)] once the 15 callsites migrate"). The user's prompt for Plan 02 also lists this as a success criterion: *"After migration, the #[allow(dead_code)] from Plan 01's nono_home_dir() and warn_once_test_home() is removed (helper is now a live consumer)"*. Plan 02's task action did not enumerate this as an explicit edit, but the success criterion mandates it; CLAUDE.md "Lazy use of dead code" further reinforces removing the attribute once a function is genuinely used.
- **Fix:** Removed both `#[allow(dead_code)]` attributes from `crates/nono-cli/src/config/mod.rs`. Both functions now have live consumers across 11 source files (verified by Tasks 1+2 grep gates).
- **Files modified:** `crates/nono-cli/src/config/mod.rs`
- **Verification:** `cargo build -p nono-cli` clean (no `dead_code` warning, confirming both functions are reachable).
- **Committed in:** `83b837c2` (Task 3 commit)
