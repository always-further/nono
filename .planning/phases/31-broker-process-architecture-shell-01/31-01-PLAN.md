<!-- scripts/test-windows-shell-write-deny.ps1 lines 121-137 — false-PASS bug to be fixed -->
```powershell
$injected = @"
try {
  Out-File '$targetFile' 'phase 30 write-deny test' -ErrorAction Stop  # <-- BROKEN: invalid Out-File syntax
} catch {
  Write-Host "[harness] Out-File threw: `$_"
}
if (Test-Path '$targetFile') { exit 1 } else { exit 42 }
"@
```

Bug: `Out-File 'path' 'content'` is not valid PowerShell — `Out-File` takes `-FilePath` (or pipeline input) plus `-InputObject`, NOT positional args in that order. Without `-FilePath`, PowerShell parses the second positional as the encoding parameter. The result is that `Out-File` ALWAYS throws (parse-time, not runtime), `catch` ALWAYS runs, the file is NEVER created, and `exit 42` (PASS sentinel) ALWAYS fires — masking whether the write was OS-denied or never attempted. RESEARCH Open Q3 / 30-WAVE-2-PROCMON.md.

Fix: `Set-Content -Path '$targetFile' -Value 'phase 31 write-deny test' -ErrorAction Stop`. With `-ErrorAction Stop`, an OS-level `UnauthorizedAccessException` from MIC NO_WRITE_UP raises, the `catch` runs, the file does NOT exist, exit 42 (PASS) fires correctly. If MIC is broken, `Set-Content` succeeds, the file exists, `exit 1` (FAIL) fires.
</harness_fix_excerpt>

```rust
/// RAII wrapper around a Win32 HANDLE — closes via `CloseHandle` on Drop exactly once.
///
/// Used by `create_low_integrity_primary_token` (Phase 31 D-06 lift) and any
/// downstream consumer that needs token handle ownership without manually
/// auditing every error path. The wrapper is `pub` so `nono-shell-broker`
/// (Phase 31 D-05) and `nono-cli` consume the same type.
///
/// Null handles are skipped on Drop (idiomatic Win32: zero handle = no-op).
#[cfg(target_os = "windows")]
#[derive(Debug)]
pub struct OwnedHandle(pub HANDLE);

#[cfg(target_os = "windows")]
impl OwnedHandle {
    /// Returns the raw `HANDLE` for FFI use without transferring ownership.
    pub fn raw(&self) -> HANDLE {
        self.0
    }
}

#[cfg(target_os = "windows")]
impl Drop for OwnedHandle {
    fn drop(&mut self) {
        if !self.0.is_null() {
            unsafe {
                // SAFETY: This handle is owned by the wrapper and is closed
                // exactly once on drop; null was checked above.
                let _ = CloseHandle(self.0);
            }
        }
    }
}
```

#[cfg(all(test, target_os = "windows"))]
mod create_low_integrity_primary_token_tests {
    use super::{create_low_integrity_primary_token, OwnedHandle};
    use windows_sys::Win32::Foundation::HANDLE;
    use windows_sys::Win32::Security::{
        GetSidSubAuthority, GetSidSubAuthorityCount, GetTokenInformation, TokenIntegrityLevel,
        TOKEN_MANDATORY_LABEL,
    };
    use windows_sys::Win32::System::SystemServices::SECURITY_MANDATORY_LOW_RID;

    /// Phase 31 D-06: library-side guarantor that the lifted function still
    /// produces a token at Low IL (RID 0x1000). Mirrors the assertion that
    /// nono-cli's `low_integrity_primary_token_sets_low_il` made on the
    /// pre-lift code; this version proves the new home crate works.
    #[test]
    #[allow(clippy::unwrap_used)]
    fn create_low_integrity_primary_token_returns_low_il_token() {
        let token = create_low_integrity_primary_token()
            .expect("create_low_integrity_primary_token must succeed in a normal user-logon test process");
        assert!(!token.0.is_null());

        // Two-call GetTokenInformation pattern: probe size, then fetch.
        let mut needed: u32 = 0;
        unsafe {
            // SAFETY: Probe call with null buffer is the documented size-discovery pattern.
            GetTokenInformation(token.0, TokenIntegrityLevel, std::ptr::null_mut(), 0, &mut needed);
        }
        let mut buf = vec![0u8; needed as usize];
        let ok = unsafe {
            // SAFETY: buf is sized by the probe call above; token.0 is a valid HANDLE.
            GetTokenInformation(token.0, TokenIntegrityLevel, buf.as_mut_ptr() as *mut _, needed, &mut needed)
        };
        assert!(ok != 0);

        // SAFETY: GetTokenInformation populated buf with a TOKEN_MANDATORY_LABEL prefix.
        let label = unsafe { &*(buf.as_ptr() as *const TOKEN_MANDATORY_LABEL) };
        let count = unsafe { *GetSidSubAuthorityCount(label.Label.Sid) };
        let last = unsafe { *GetSidSubAuthority(label.Label.Sid, (count - 1) as u32) };
        assert_eq!(
            last,
            SECURITY_MANDATORY_LOW_RID as u32,
            "duplicated token must be at Low integrity (RID 0x1000); got 0x{last:x}"
        );
    }

    /// Phase 31 D-06: validate OwnedHandle::Drop does not panic / double-close.
    /// Exercises the lifted RAII wrapper directly.
    #[test]
    fn owned_handle_drop_is_safe_for_low_il_token() {
        let token = create_low_integrity_primary_token()
            .expect("create_low_integrity_primary_token must succeed");
        assert!(!token.0.is_null());
        drop(token); // Must NOT panic, abort, or double-close.
    }
}
