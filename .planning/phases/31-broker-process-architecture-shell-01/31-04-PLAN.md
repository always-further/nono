<objective>
Extend the Windows release pipeline so `nono-shell-broker.exe` ships alongside `nono.exe` as an Authenticode-signed sibling artifact, satisfying D-05 (broker artifact in releases) and D-07 (broker resolves as sibling of nono.exe via `current_exe().parent()`). The pipeline must:

1. Build the broker on every Windows release run via the existing `cargo build` infrastructure (added to the existing build step OR as a sibling step matching the same target triple).
2. Sign the broker with the same Authenticode key as `nono.exe` so the Phase 28 `parse_signer_subject` chain-walker (`AUDC-01..03`) records a consistent signer for any audit-ledger trace involving the broker process. Reuses `scripts/sign-windows-artifacts.ps1` verbatim — only the `-ArtifactPaths` array passed in extends.
3. Verify the broker's Authenticode signature via the existing dual-check pattern (`signtool verify /pa /tw` inside `sign-windows-artifacts.ps1` + secondary `Get-AuthenticodeSignature` after) and FAIL the release if either check fails (D-13 fail-closed).
4. Include the broker in both machine-scope and user-scope MSI installers via `scripts/build-windows-msi.ps1` so install-time deployment lands `nono-shell-broker.exe` next to `nono.exe` (the install dir IS the parent dir `current_exe()` resolves to at runtime).
5. Upload the broker alongside `nono.exe` in the GitHub release: as a standalone signed artifact (`nono-shell-broker-<tag>-x86_64-pc-windows-msvc.exe`) AND inside the existing zip/MSI payloads. Match the existing artifact-staging pattern at `release.yml:225-241`.

The plan does NOT modify the signing primitive itself (`scripts/sign-windows-artifacts.ps1`'s signtool invocation is byte-identical) — only the calling shape of release.yml extends.

Purpose: At Plan 31-05 field-test time, the user's Windows test box must have `nono-shell-broker.exe` installed alongside `nono.exe` (whether via MSI install or via zip extraction). Without this plan, Plan 31-03's `BrokerLaunch` arm fails with `NonoError::BrokerNotFound` 100% of the time. This plan closes the deployment loop.

Output: Tag-triggered release on Windows produces a signed `nono-shell-broker.exe` available as: (a) standalone artifact, (b) in the zip bundle next to nono.exe, (c) in both MSI installers. Plan 31-05's field-test runs against an installed/zip-extracted `nono.exe` that finds its sibling broker.
</objective>
