<objective>
Add the `WindowsTokenArm::BrokerLaunch` cascade arm to `crates/nono-cli/src/exec_strategy_windows/launch.rs` and dispatch it for the PTY+supervised launch path. The new arm:

1. Resolves `nono-shell-broker.exe` as a sibling of `std::env::current_exe()` (D-07; no env-var override surface — env-poisoning attack rejected).
2. Constructs the broker command-line per Plan 31-02's argv contract: `--shell <path> --shell-arg <arg>... --inherit-handle <hex>... --cwd <path>` (D-08).
3. Builds a `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` whitelisting ONLY the ConPTY input/output pipe handles for inheritance through the broker boundary; capability-pipe handles (Phase 11) and any other supervisor handles are NOT inheritable past `nono.exe` (D-02).
4. Spawns the broker via `CreateProcessW` (broker runs at caller's identity = Medium IL; broker self-degrades) instead of `CreateProcessAsUserW(low_il_token, ...)` directly (D-15: replaces the direct Low-IL spawn that triggered `0xC0000142` on the supervised+ConPTY path).
5. Calls `apply_process_handle_to_containment(containment, broker_process.raw())` BEFORE `ResumeThread(broker)` so the broker (and its child via Win32 cascade) is inside the Job Object; `JOB_OBJECT_LIMIT_*BREAKAWAY*` flags remain unset (D-04).
6. Returns the broker's `WindowsSupervisedChild` to existing supervisor plumbing (D-03: no new IPC; broker→child→exit-code chain runs through existing handle).
7. Rewrites `pty_token_gate_tests` (6/6) for the new dispatch and adds a new unit test verifying `BrokerNotFound` returns when the sibling is absent. Adds a Windows-only Job Object containment test verifying `IsProcessInJob` reports the spawned child is in the Job (D-04 acceptance).

Per D-15 conditional: the planner verifies the Direct path (`pty_none_caps_demand_low_il_selects_low_il`) is still structurally reachable in CI / future code. Verification: today the cascade reaches `LowIlPrimary` only when `caps_demand_low_il=true && has_session_sid=false`; on Windows supervised launches `session_sid` is unconditionally `Some(...)` per `execution_runtime.rs:334` (the cascade never reaches the LowIlPrimary fallback today). HOWEVER, the Phase 30 unit test that asserts `LowIlPrimary` was added explicitly to pin the helper's behavior for "future readers and for non-Windows platforms where the helper compiles cleanly" (`launch.rs:1564-1574`). Decision: KEEP `WindowsTokenArm::LowIlPrimary` variant + the `pty_none_caps_demand_low_il_selects_low_il` test (asserting `LowIlPrimary`) + the `low_integrity_primary_token_tests` Windows-FFI module — the variant is harmlessly preserved as a fallback, and removing it would delete the only direct runtime exercise of the lifted FFI function. This satisfies the D-15 fallback clause ("If the planner finds a Direct path that still needs Low-IL spawn, re-evaluate D-15 — keep LowIlPrimary as a fallback arm and only rewrite the PTY-supervised tests").

Output: A green-build cascade with `BrokerLaunch` as the live PTY+supervised arm. Plan 31-05 field-test on the user's Windows test box exercises the dispatch end-to-end against Acceptance #1-#7.

Plan 31-03 constructs the broker command line as:
```
"<broker_exe>" --shell "<launch_program>" --shell-arg "<arg1>" --shell-arg "<arg2>" ... \
   --inherit-handle 0x<conpty_input_pipe_hex> --inherit-handle 0x<conpty_output_pipe_hex> \
   --cwd "<current_dir>"
```

Where:
- `<broker_exe>` = `current_exe.parent()` + `nono-shell-broker.exe`.
- `<launch_program>` = the existing `launch_program: &Path` argument to `spawn_windows_child` (line 1171).
- `<arg1>..<argN>` = the existing `cmd_args: &[String]` argument (line 1173).
- `<conpty_input_pipe_hex>` / `<conpty_output_pipe_hex>` = read from `pty_pair` struct fields (executor reads the struct def for exact field names; likely `pty_pair.input_handle` / `pty_pair.output_handle` or similar).
- `<current_dir>` = the existing `config.current_dir` argument.

Each handle is formatted as a hex string with leading `0x` for clarity: `format!("0x{:016x}", handle as usize)` on x64. The broker's parser strips `0x` and uses `usize::from_str_radix(_, 16)`.

Quoting: every argument that contains whitespace, `"`, or other special characters MUST be quoted via the existing `build_command_line` helper at `launch.rs` (search for `build_command_line` to find the helper). If `build_command_line` does NOT support a "list of pre-quoted args" mode, the executor adds a small helper `build_broker_command_line(broker_exe: &Path, broker_args: &[OsString]) -> Vec<u16>` that mirrors `build_command_line`'s quoting rules.

The new dispatch arm REPLACES the existing PTY block at `launch.rs:1257-1349`. Differences from the existing `PROC_THREAD_ATTRIBUTE_PSEUDOCONSOLE` block:

| Aspect | Existing (PSEUDOCONSOLE) | New (BrokerLaunch + HANDLE_LIST) |
|--------|--------------------------|----------------------------------|
| Attribute name | `PROC_THREAD_ATTRIBUTE_PSEUDOCONSOLE` | `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` |
| Attribute value | `pty_pair.hpcon` (single HPCON) | Array of inheritable HANDLEs (ConPTY input + output pipes) |
| Attribute value byte size | `size_of::<HPCON>()` | `size_of_val(&handles_array[..])` |
| CreateProcess function | `CreateProcessAsUserW(h_token, ...)` (Low-IL token) | `CreateProcessW(...)` (broker runs at caller's identity = Medium IL) |
| dwCreationFlags | `CREATE_SUSPENDED \| CREATE_UNICODE_ENVIRONMENT \| EXTENDED_STARTUPINFO_PRESENT` | Same: `CREATE_SUSPENDED \| CREATE_UNICODE_ENVIRONMENT \| EXTENDED_STARTUPINFO_PRESENT` |
| bInheritHandles | 0 | 1 (HANDLE_LIST gates the actual inherited set) |
| ConPTY ownership | nono.exe owns HPCON; passes via PSEUDOCONSOLE attr | nono.exe still owns HPCON; broker just inherits the pipe handles via HANDLE_LIST. Broker does NOT call CreatePseudoConsole. |

The `Init/Update/Delete ProcThreadAttributeList` shape is identical (Pattern S-04 from PATTERNS.md):

```rust
let mut attr_size: usize = 0;
unsafe { InitializeProcThreadAttributeList(std::ptr::null_mut(), 1, 0, &mut attr_size); }
let mut attr_buf = vec![0u8; attr_size];
let attr_list = attr_buf.as_mut_ptr() as LPPROC_THREAD_ATTRIBUTE_LIST;
let ok = unsafe { InitializeProcThreadAttributeList(attr_list, 1, 0, &mut attr_size) };
// ...

let inherit_handles: [HANDLE; 2] = [pty_pair.input_handle, pty_pair.output_handle];  // executor verifies field names
let ok = unsafe {
    UpdateProcThreadAttribute(
        attr_list, 0,
        PROC_THREAD_ATTRIBUTE_HANDLE_LIST as usize,
        inherit_handles.as_ptr() as *mut _,
        std::mem::size_of_val(&inherit_handles),
        std::ptr::null_mut(), std::ptr::null_mut(),
    )
};
// ... CreateProcessW (NOT AsUserW) with EXTENDED_STARTUPINFO_PRESENT, bInheritHandles=1 ...
unsafe { DeleteProcThreadAttributeList(attr_list); }
```

The executor MUST also flip the ConPTY pipe handles to `bInheritable=TRUE` BEFORE the `CreateProcessW` call (using `SetHandleInformation(handle, HANDLE_FLAG_INHERIT, HANDLE_FLAG_INHERIT)`) and flip them BACK to non-inheritable AFTER (so they don't accidentally leak into other CreateProcess calls in the supervisor). HANDLE_LIST + bInheritHandles=1 gates which inheritable handles cross the boundary; only handles marked inheritable AT THE TIME OF CreateProcessW are eligible.

```rust
/// Build a Win32 command line for spawning nono-shell-broker.exe with flat argv.
/// Quoting matches the D-08 contract: shell paths and args containing whitespace
/// or quote characters are double-quoted with embedded quotes doubled.
fn build_broker_command_line(broker_exe: &Path, args: &[std::ffi::OsString]) -> Vec<u16> {
    let mut cmd = String::new();
    cmd.push('"');
    cmd.push_str(&broker_exe.to_string_lossy());
    cmd.push('"');
    for a in args {
        cmd.push(' ');
        let s = a.to_string_lossy();
        if s.contains(' ') || s.contains('"') {
            cmd.push('"');
            cmd.push_str(&s.replace('"', "\"\""));
            cmd.push('"');
        } else {
            cmd.push_str(&s);
        }
    }
    OsStr::new(&cmd).encode_wide().chain(Some(0)).collect()
}
```
