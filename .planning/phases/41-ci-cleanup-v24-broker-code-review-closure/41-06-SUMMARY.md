## Plan 31-02 SUMMARY Supersession Note

Plan 31-02 SUMMARY claimed "empty `--inherit-handle` list = most-restrictive (expected) shape — the spawned child inherits NO handles." This claim is now **obsolete**.

With CR-03, the broker argv parser **rejects** an empty `--inherit-handle` list at the parse boundary with a `SandboxInit` error. The empty-list path is correct-by-construction-rejected, not correct-by-runtime-error. This eliminates the ambiguity of whether `UpdateProcThreadAttribute(PROC_THREAD_ATTRIBUTE_HANDLE_LIST, cbSize=0)` succeeds on all Windows versions.

Per project convention, archived SUMMARYs in `.planning/phases/.archive/` are **not back-edited**. The supersession is documented here instead.

## nono.h Regeneration Note

The plan's acceptance criterion referenced "cbindgen regenerating nono.h to reflect the D-09 remap from the match-arm doc-comment." This was partially incorrect — cbindgen does not include `pub(crate)` implementation comments from `map_error`. The correct cbindgen-reachable surface is the `types.rs` enum variant doc-comment.

**Resolution applied:** Updated the `ErrSandboxInit` variant doc-comment in `bindings/c/src/types.rs` to enumerate `BrokerNotFound` as a new mapping source (D-09 CR-01). This caused cbindgen to regenerate `nono.h` with 5 new lines in the `NONO_ERROR_CODE_ERR_SANDBOX_INIT` comment block. `git diff bindings/c/include/nono.h` is non-empty, confirming successful regen.
