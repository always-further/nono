**Site 1 — `crates/nono-cli/src/exec_strategy/supervisor_linux.rs:1424`** (inside `cgroup_session_pre_exec_places_pid` test, body lines 1415-1440):

```rust

        fn cgroup_session_pre_exec_places_pid() {
            let Some(session) = try_cgroup_session("test-pre-exec-001") else {
                return;
            };
            session.apply_limits().unwrap();
            let mut cmd = std::process::Command::new("sleep");

            let child_pid = child.id();
```

**CLAUDE.md § Unwrap Policy violation:** `cmd.spawn().unwrap()` AND `session.apply_limits().unwrap()` are forbidden under `clippy::unwrap_used`. Also, `let child = cmd.spawn().unwrap();` followed by drop without explicit `child.wait()` triggers `clippy::zombie_processes` because the `Child` handle is dropped while the OS-level process is still running.

**Site 2 — `crates/nono-cli/src/exec_strategy/supervisor_linux.rs:1451`** (inside `cgroup_kill_terminates_grandchildren` test, body lines 1442-1460):

```rust

            let Some(session) = try_cgroup_session("test-kill-001") else {
                return;
            };
            session.apply_limits().unwrap();
            let mut cmd = std::process::Command::new("bash");
            cmd.args(["-c", "for i in 1 2 3; do sleep 60 & done; wait"]);
            session.install_pre_exec(&mut cmd);
            let mut child = cmd.spawn().unwrap();                      // ← LINE 1451
            // Give bash time to fork the sleep children.
            std::thread::sleep(std::time::Duration::from_millis(200));
            session.kill_all().unwrap();

            );

```

This test ALREADY calls `child.wait().unwrap()` at line 1455 (the `let result = child.wait().unwrap();` line), so the zombie risk is mitigated for THIS test — but `.unwrap()` violations remain at lines 1447, 1451, 1454, 1455. The Site 1 test has NO `wait()` call at all (it relies on `session.kill_all()` to terminate the process via cgroup signal), so `clippy::zombie_processes` fires on the dropped `child` handle.

**Fix approach (executor picks within bounded options):**

OPTION A (cleanest): Change both test fn signatures from `fn cgroup_session_pre_exec_places_pid()` to `fn cgroup_session_pre_exec_places_pid() -> Result<(), Box<dyn std::error::Error>>` and replace every `.unwrap()` with `?`. For the zombie issue in Site 1, add `child.kill()?;` AND `child.wait()?;` before the `assert!`. This is the structurally cleanest fix.

OPTION B (minimal-touch): Add `#[allow(clippy::unwrap_used)]` to the `#[cfg(target_os = "linux")] mod tests` block — but this violates CLAUDE.md § Lazy use of dead code spirit AND existing precedent. NOT RECOMMENDED.

OPTION C (per-call .expect): Replace `.unwrap()` with `.expect("...")` — but CLAUDE.md § Unwrap Policy treats `.expect()` as equivalently forbidden. NOT RECOMMENDED.

Use OPTION A. The mod is `#[cfg(target_os = "linux")] mod tests {` so the changes are Linux-only.
