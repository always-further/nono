### Deviation Verification: D-36.5-C2-DEV1 (reserved_profile_source inlines would_shadow_builtin)

**Semantic equivalence confirmed:**

`reserved_profile_source` at `profile_cmd.rs:404-426` and `would_shadow_builtin` at `profile_save_runtime.rs:232-242` share identical logic for the built-in shadow check:

1. Both call `profile::is_user_override(name)` → return `false` if user override exists (explicit update, not new shadow)
2. Both call `crate::policy::load_embedded_policy()` → `policy.profiles.contains_key(name)` on success
3. Both return `true` (fail-closed) on `Err(_)` — "refuse rather than risk silently shadowing a built-in we couldn't enumerate"

`reserved_profile_source` additionally checks `get_package_for_profile` for package-managed profiles, which is an extension beyond `would_shadow_builtin` (more restrictive, not less). The deviation is correctly described and security-correct: fail-closed posture is preserved on the Windows path.
