## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| caller -> `cmd_verify` / `cmd_show` | CLI user-supplied `<session_id>` argument; `validate_session_id` is the choke point against path-injection (audit_session.rs:289-303). | session ID string (untrusted input) |
| `load_session` filesystem read | Trusts `audit_root()` and `rollback_session::rollback_root()` returned via `nono_home_dir()` (Phase 27.1 seam). Canonicalize-then-`Path::starts_with` re-establishes trust post-symlink-resolution. | filesystem paths (resolved) |
| `tracing::warn!` emission | Writes to global tracing subscriber routed to stderr (cli_bootstrap.rs:107, 115). | path-display strings, no secrets |
| `create_audit_state` filesystem writes | Trusts `nono_home_dir()` and `rollback_destination`; both audit and rollback dirs created with `0o700` on Unix. | session directory paths |
| Bundle write target | Bundle now lives at `<audit_root>/<id>/`. Audit root is append-mostly; rollback root is mutable. New layout protects bundle from rollback-root mutability. | DSSE-signed audit-attestation.bundle |
| Cleanup boundary | `rm -rf <rollback_root>/<id>/` no longer destroys the bundle (D-27.2-01 closes the non-repudiation hole). | non-repudiation evidence |
| Test parent process -> spawned `nono` subprocess | Test threads `NONO_TEST_HOME` via `Command::env`; subprocess inherits a controlled tempdir. | tempdir path, NONO_TEST_AUDIT_KEY_VERIFY env var |
| Test parent process -> host env vars | `ScopedEnvVar` mutates parent process env; per-invocation `{pid}_{nanos}` suffix + RAII Drop-restore minimize blast radius. | test secret string (deterministic, not real credential) |
| Subprocess output -> test assertions | `serde_json::from_slice` on subprocess stdout is the parsing boundary. Stdout purity guaranteed by tracing-to-stderr routing (T-27.2-DEV-01). | flat-shape `audit verify --json` output |
